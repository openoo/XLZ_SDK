# 设计说明

## 借鉴点

- 借鉴 CornerstoneSDK 的 C++ 分层：API、框架数据类型、易语言基础类型、工具函数分开。
- 借鉴 CornerstoneSDK 对易语言数组的谨慎处理，不再把返回数组直接声明成 `std::vector`。
- 借鉴 XiaoLiZi_SDK 的权限安全标记思路，`AppInfo` 对高风险权限写入 `state/safe=0`。
- 不照搬旧版本 API 表，而是以本地 `cornsdk5.1.7_免打包版` 解析结果生成完整元数据。

## 稳定性约束

- 函数地址统一使用 `std::uintptr_t`，避免指针截断。
- 小栗子 API 指针统一按 `NativeFunction` 调用，第一个参数固定为 `pluginkey`。
- C++ `bool` 传给易语言逻辑型前使用 `ToEBool` 转成 32 位整数。
- 返回 `const char*` 的 SDK 自有函数使用线程局部缓存，避免返回局部对象地址。

## 源码分层

- `src/api.cpp`：核心地址解析和基础调用，不放具体业务 API。
- `src/api_convenience.cpp`：少量高频、人工核对过参数顺序的便捷函数。
- `include/xlz/generated_api.hpp`：由元数据生成的全量 API 包装，覆盖全部 434 个公开 API。
- `src/generated_api.cpp`：全量包装的编译锚点和数量一致性断言。
