# XLZ_SDK 开发指南

## 设计目标

XLZ_SDK 是面向小栗子框架的现代 C++ SDK。当前版本以 `cornsdk5.1.7_免打包版` 为数据源，重点保证 API 名称、参数顺序、返回类型和传址标记尽量还原原始 SDK，同时避免把易语言数组误当成 C++ 容器。

## 目录结构

- `include/xlz/`：对外公开头文件。
- `src/`：SDK 实现文件。
- `examples/`：插件示例和通用调用示例。
- `docs/`：中文开发文档与 API 清单。
- `tools/`：元数据一致性检查工具。

## API 分层

- `xlz::Api` 是核心调用器，`src/api.cpp` 只负责初始化后的函数地址查找和 `CallRaw/CallText/CallBool` 等基础调用。
- `src/api_convenience.cpp` 放少量人工确认过的高频便捷封装，例如发送消息、取框架 QQ、登录/下线 QQ。
- `xlz::GeneratedApi` 是自动生成的全量包装器，包含 434 个 `Api001...Api434` 方法，注释里保留中文 API 名、返回类型、参数顺序和传址标记。

## 调用 API

```cpp
api.SendGroupMessage(botQQ, groupQQ, "hello");
api.CallText("发送群消息", botQQ, groupQQ, "hello", xlz::ToEBool(false));
```

## 传址参数

`参考`、`传址`、`取变量地址(...)` 对应 C++ 指针：

```cpp
std::int64_t random = 0;
std::int32_t req = 0;
api.SendPrivateMessage(botQQ, friendQQ, "可撤回消息", &random, &req);
```

## 易语言数组

易语言动态数组不是 `std::vector`。SDK 使用 `xlz::EArrayView<T>` 表示框架返回的数组视图，不负责释放内存；需要长期保存时先复制。

## 编译

小栗子常见宿主是 32 位进程，插件 DLL 必须与宿主位数一致。CMake 已统一 MSVC 运行库为静态 `/MT`，减少部署时缺少 VC 运行库的概率。

```powershell
cmake -S . -B build -A Win32
cmake --build build --config Release
```

64 位宿主可改用：

```powershell
cmake -S . -B build-x64 -A x64
cmake --build build-x64 --config Release
```

## 插件初始化兼容性

小白兔对 `appload` 返回 JSON 的格式校验较严格。SDK 的 `xlz::AppInfo` 会生成与已验证可加载 C++ 插件一致的结构：

```json
{
  "data": {
    "needapilist": {
      "输出日志": {
        "desc": "插件所需权限说明"
      }
    }
  }
}
```

兼容性要求：

- 示例插件只导出 `appload`，避免宿主误取其他初始化入口。
- `needapilist` 的每个权限对象只写 `desc` 字段。
- `appload` 返回文本会从 SDK 内部 UTF-8 转为 ANSI/GBK，匹配易语言宿主的文本编码。
- 示例插件只申请实际用到的最小权限，避免老宿主无法识别新版本全量权限名。
- 32 位小白兔只能加载 Win32 DLL；x64 DLL 会在 `LoadLibraryA` 阶段失败。
